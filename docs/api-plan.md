## 사용자 권한

1. 소유자 (unique + 최고 권한 = 관리자 모든 권한 + 관리자는 소유자 편집 불가)
2. 관리자 (편집자 권한 + 관리자 이하 단계 사용자의 권한+프로필 편집 가능)
3. 편집자 (대시보드 차트 조회 및 편집 권한 보유, 다른 사용자의 권한 수정 불가)
4. 뷰어 (조회 권한만 보유)

## 3. 대시보드/페이지

- **페이지** = 차트, 표, 각 요소의 픽셀 상 위치, 데이터 참조 등 하나의 페이지를 구성하는 요소 및 설정 등 종합
- **대시보드** = 여러 개의 페이지 모음
- 페이지 및 대시보드는 생성한 사용자별로 소유권을 가짐
- 편집자 끼리는 소유권이 없는 페이지 및 대시보드 수정 불가, 관리자는 가능
- 대시보드 또는 페이지 즐겨찾기 가능
- 현재 대시보드에서 다른 페이지 추가 또는 다른 대시보드로 덮어쓰기 가능
- 페이지 크기는 픽셀 단위로 지정, 페이지 내 요소들은 1px 단위로 위치 지정

## 4. 데이터 소스

- 외부 DB로부터 SQL 쿼리 기반 데이터 소스 불러오기
- 또는 엑셀 파일 등을 업로드하여 데이터 소스 추가
- 데이터 소스 추가 및 편집은 관리자 이상만 가능
- 데이터 소스의 필드에는 `{데이터 소스 ID(영문 테이블명 등)}.{필드 ID(영문 칼럼명 등)}` 이렇게 접근
- 데이터 소스의 필드에 한글 등으로 라벨 추가 가능, 대시보드에서는 ID 대신에 라벨로 표시
- 데이터 소스도 한글 등으로 라벨 추가 가능, 대시보드에서는 ID 대신에 라벨로 표시
- 데이터 소스 필드별로 유형 관리 (텍스트, 숫자 등)
- 데이터 소스 필드별로 기본 집계 단위 설정 가능 (합계, 평균, 최솟값, 최댓값 등)
- 데이터 소스 필드별로 설명 문구 추가 가능
- 데이터 소스를 참조하는 대시보드, 페이지로 이동하는 기능
- 관리자는 데이터 소스에 커스텀 그룹 또는 특정 사용자 단위로 접근 권한 설정
- 데이터 소스 권한이 없는 사용자는 데이터 소스를 참조하는 대시보드, 페이지에 접근 불가

## 5. 차트

- 차트는 시계열, 막대, 표, 피벗 등이 있을 수 있음
- 차트는 프론트엔드에서 구성하지만, 이 차트를 구성하기 위한 설정 및 데이터 제공은 백엔드 및 DB에서 관리
- 차트는 페이지에 귀속되며 1px 단위로 가로, 세로 위치 및 너비, 높이 지정
- UI에서는 차트에 단일 데이터 소스를 선택하고 필드 목록을 선택해 추가 가능
- 필드 항목은 행 단위로 표시하거나, 측정기준(dimensions) 기준으로 측정항목(metrics) 를 집계 가능
- 측정항목 집계 시 기본 집계 단위 또는 직접 합계, 평균, 최솟값 등 지정 가능
- 필드 항목 단위로 정렬 기능
- 차트 스타일 설정 (헤더 글꼴, 헤더 줄바꿈, 색상, 라벨 글꼴, 데이터 누락 표시, 항목별 히트맵, 배경 및 테두리 등)
